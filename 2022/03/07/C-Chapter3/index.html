C++ Primer第三章 字符串、向量、数组 学习笔记

3.1 标准库 using声明

之前使用过的cin cout都属于标准库std里面的库函数，需要显式的使用域操作符std::cin来标识命名空间std，这种方式略显繁琐。在实际使用中可以直接使用using声明在开头标识出所需的命名空间

using namespace::name如 using std::cin。

头文件不应当包含using声明，如果头文件包含了using声明，那么使用了该头文件的全部文件都会拥有该名字，可能会产生始料未及的命名冲突。

3.2 String

3.2.1 直接初始化与拷贝初始化

使用等号=进行的初始化叫做拷贝初始化，编译器将等号右边的初始值拷贝到新对象中去。反之，不使用等号，执行的是直接初始化。

3.2.4 size与string::size_type

s.size();的功能是返回s中字符的数量，则size返回的一定是一个非负整数。但size返回的类型并不是简单的int或者unsigned类型 ，而是特殊的string::size_type类型的值。string类和其它大多数标准库类型都定义了几种配套的类型，它们体现了标准库类型与机器无关的特性。

由于size函数返回的是一个无符号的整型数，在表达式中不能简单地与int混用，否则会出现意想不到的错误。

3.2.6 使用下标

想要访问string对象的单个字符时，需要使用到string的下标。下标运算符[]接收的输入参数的类型是string::size_type的值。

因此想要使用下标执行迭代，下标应当声明成string::size_type类型。

3.3 标准库类型Vector

vector类型表示对象的集合，其中所有对象的类型均相同。集合内的对象可以按索引寻址。

3.3.1 vector的声明

需要声明使用vector标准库

#include<vector>

3.3.2 vector的添加

vector类型具有成员函数push_back，作用是将一个元素当成vector的尾元素压栈进vector，因此vector可以方便的动态增长。

3.4 迭代器

C++中的容器如Vector等和不属于容器的string对象都可以使用迭代器来间接访问对象元素。

3.4.1 迭代器的使用

迭代器通常通过两个成员来使用：begin和end。begin成员负责返回指向第一个元素的迭代器，end成员则负责返回指向容器尾元素的下一位置的迭代器。可以理解成数据结构队列中的队首指针和队尾指针。

++iter令迭代器指示容器中的下一个元素，解引用返回迭代器的元素的引用。因为end迭代器并不指向一个实际存在的元素，因此并不能对其进行递增和解引用。

例：将string对象中的第一个单词改写成大写形式

3.4.2 迭代器类型

迭代器成员begin和end的返回的具体类型由其指向的对象是否是常量决定。如果对象是常量，begin和end返回的类型是 const_iterator，如果对象不是常量返回 iterator。

但如果需要的迭代器对象只需要读操作而无需写操作，我们最好使用常量迭代器类型。使用成员cbegin和cend来获得const_iterator类型的返回值。

auto it3 = v.cbegin();

3.4.3 解引用和成员访问

解引用迭代器可以获得迭代器所指的对象，但是如果获得的对象是类的对象，可能需要进一步获得类的成员。因此定义了箭头运算符直接获得对象的成员。

凡是使用了迭代器的循环体，如范围for循环，都不要向迭代器所属的容器中添加元素。

3.5 数组

数组也是存放一组相同类型对象的容器，但是数组的大小确定不变，不能随意向数组中增加元素。

3.5.1 数组的特殊要求

1、字符数组存储字面值会多保存一个空字符。

字符串字面值的结尾有一个空字符，使用字符串字面值对字符数组进行拷贝初始化，这个空字符也会被拷贝到字符数组中去。

2、数组不允许拷贝和用数组赋值。

3.5.2 数组的访问

使用数组下标时，通常将下标类型定义为size_t类型，size_t是一种机器相关的无符号类型，它足够表示内存中任意对象的大小。

3.5.3 指针与数组

想要获取数组的首指针和尾指针，可以使用begin和end函数，但是与vector\string等容器的迭代器的使用不同的是，数组对象的begin\end并不是成员函数，需要将数组作为他们的参数

指针的解引用和指针运算

指针加上一个整数还是指针，若它指向了一个元素，则允许解引用该指针

数组的名字通常是一个指向数组首元素的指针。
