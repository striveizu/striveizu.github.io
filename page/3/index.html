3.2 值类型和引用类型

所有像 int、float、bool 和 string 这些基本类型都属于值类型，使用这些类型的变量直接指向存在内存中的值：

值类型变量的值存储在堆中。

更复杂的数据通常会需要使用多个字，这些数据一般使用引用类型保存。

一个引用类型的变量 r1 存储的是 r1 的值所在的内存地址（数字），或内存地址中第一个字所在的位置。

同一个引用类型的指针指向的多个字可以是在连续的内存地址中（内存布局是连续的），这也是计算效率最高的一种存储形式；也可以将这些字分散存放在内存中，每个字都指示了下一个字所在的内存地址。

当使用赋值语句 r2 = r1 时，只有引用（地址）被复制。

如果 r1 的值被改变了，那么这个值的所有引用都会指向被修改后的内容，在这个例子中，r2 也会受到影响。

解题思路

之前考研复习递归算法，常见的递归应用都是二叉树，比如二叉树的前序、中序、后序遍历。一个直觉的想法是递归函数完成的是一个确定的功能，将问题分解成不可再分的子问题，再俄罗斯套娃似的将子问题的答案作为上一层调用的参数。

比如二叉树的中序遍历函数 inorderTraversal(TreeNode* root)：完成的是对以root为根对二叉树遍历，直到叶子节点对节点进行访问。

在本题中， ListNode reverseList(ListNode head)实现的是将以head为头节点的链表反转并返回新的头节点。

在本题中，递归反转整个链表，重点是链表的反转，即把后一个元素的next域连接到前一个元素上，并且将前一个元素的next域断掉置空（不置空可能出现链表有环）。

因此我们可以通过使用head.next来遍历链表，并将其作为参数传入递归函数。

首先检查边界条件（力扣中的题必有边界检查，如空链表、只有一个元素的链表），这种直接返回头节点就好。

然后递归、后一个元素的next域连接到前一个元素上，并且将前一个元素的next域断掉置空

由于我们最后需要返回新链表的头节点，但此时head的next域置空了，因此我们需要一个变量来保存最后一个元素。

1 常见指针思路

在链表的题目中，最常见的题型如链表合并，经常会使用到指针的技巧。以合并两个有序链表为例，通过使用两个指针分别遍历两个链表，依次将链表上的元素摘下连接到新表即可实现有序合并。

1.1 使用优先级队列找到最小元素

1.1.1 #23 合并K个有序链表

题目链接：23. 合并K个升序链表 - 力扣（LeetCode） (leetcode-cn.com)

力扣的第23题是合并K个有序链表，题目如下：

题目分析：

链表组成一个数组，假设数组中包含k个链表，创建k个指针对其遍历是一种可行的办法，但是每插入一个节点都需要进行k次比较，时间复杂度极高，大约为*O(kN)**，k为链表的数量，N为这些链表的总节点数。。

我们需要按从小到大的顺序对k个有序的链表进行合并，可以使用优先级队列来依次取出k个链表中最小的那一个，再将该链表的下一个节点插入优先级队列。

时间复杂度分析：由于优先级队列的底层实现是小顶堆，小顶堆中存放的是k个链表中的当前本链表最小的节点，因此堆高为lgk，进行一次poll或者add的操作时间复杂度为O(lgk)，**总的时间复杂度为O(Nlogk)**，N为这些链表的总节点数。
