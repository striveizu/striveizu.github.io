第六章 函数

6.1 局部对象

在C++中，名字有作用域，对象有生命周期。

名字的作用域指的是程序文本的一部分，名字在其中可见。

对象的生命周期是程序执行过程中该对象存在的一段时间。

对于函数来说，函数体是一个语句块，语句块构成一个新的作用域。我们可以在作用域中定义变量，在作用域中定义的变量统称为局部变量。仅在函数的作用域内可见

6.1.1 普通局部对象

普通局部变量对应的对象只有当函数的控制路径经过变量定义语句时创建该对象，到达变量定义所在的块的末尾时将其销毁。

6.1.2 局部静态对象

由于普通局部变量的对象的生命周期在执行到变量定义所在块的末尾即结束，有时又需要让局部变量的生命周期贯穿函数调用及之后的时间，可以把局部变量定义成static类型。定义成static的对象在程序执行路径第一次经过对象定义语句时被初始化，直到程序终止才被销毁。

6.2 函数声明 又称函数原型

函数的名字必须在使用之前声明，函数只能定义一次，但是可以声明多次。

如果一个函数永不被使用，可以只有声明没有定义。

函数的声明和函数的定义非常类似，但是函数的声明无需写函数体，只需要在函数名和参数列表后加；即可。

函数的声明一般在头文件中使用，可以确保所有文件同一函数声明保持一致，并且方便修改函数的接口。

为什么要使用函数声明！！！

对于单个源文件的程序，通常是将函数定义放到 main() 的后面，将函数声明放到 main() 的前面，这样就使得代码结构清晰明了，主次分明。

使用者往往只关心函数的功能和函数的调用形式，很少关心函数的实现细节，将函数定义放在最后，就是尽量屏蔽不重要的信息，凸显关键的信息。将函数声明放到 main() 的前面，在定义函数时也不用关注它们的调用顺序了，哪个函数先定义，哪个函数后定义，都无所谓了。

然而在实际开发中，往往都是几千行、上万行、百万行的代码，将这些代码都放在一个源文件中简直是灾难，不但检索麻烦，而且打开文件也很慢，所以必须将这些代码分散到多个文件中。对于多个文件的程序，通常是将函数定义放到源文件（.c文件）中，将函数的声明放到头文件（.h文件）中，使用函数时引入对应的头文件就可以，编译器会在链接阶段找到函数体。

前面我们在使用 printf()、puts()、scanf() 等函数时引入了 stdio.h 头文件，很多初学者认为 stdio.h 中包含了函数定义（也就是函数体），只要有了头文件就能运行，其实不然，头文件中包含的都是函数声明，而不是函数定义，函数定义都放在了其它的源文件中，这些源文件已经提前编译好了，并以动态链接库或者静态链接库的形式存在，只有头文件没有系统库的话，在链接阶段就会报错，程序根本不能运行。

除了函数，变量也有定义和声明之分。实际开发过程中，变量定义需要放在源文件（.c文件）中，变量声明需要放在头文件（.h文件）中，在链接程序时会将它们对应起来

6.3 参数传递

6.3.1 形参实参

形参是函数声明时，括号里参数列表的参数。函数执行的第一步是定义并初始化它的形参。实参是形参的初始值，每次调用函数时都会重新创建它的形参，并用传入的实参对形参进行初始化。

如果形参是引用类型，那么形参会作为实参的别名与实参绑定，否则将实参的值拷贝后赋值给形参。

当形参是引用类型时，我们说它对应的实参被引用传递，或者函数被传引用调用。由于形参是实参的引用，所以函数对形参的修改都会直接作用在实参上。因此想要对外部的对象使用函数修改，需要将形参声明为引用类型。

如果形参不是引用类型，形参和实参是两个独立对象。形参只是使用实参的值拷贝赋值。函数对形参的修改不会作用在实参上。这种形式的实参被值传递，或者参数被传值调用。

6.3.2 传值参数

当初始化一个非引用类型的变量时，初始值被拷贝给变量，此时对变量的改动不会影响初始值。

当形参是一个指针类型时，初始化该指针类型变量时，与非引用类型相同执行拷贝赋值。由于形参被赋予的值是实参的值，实参也是指针类型的值，也就是一个地址。因此形参实参虽然是两个变量，但是都指向同一个地址。此时在函数中就可以使用指针来修改函数外部的对象。

6.3.3 传引用参数

由于形参不是引用类型时，会对形参进行拷贝初始化。但是拷贝大的类型或者容器对象比较低效，甚至有的类类型往往不支持拷贝操作。因此，函数的形参往往可以声明成引用类型

如果不需要对原实参进行修改，可以将形参声明成对常量的引用从而避免改变被引对象。

6.4 返回类型和return语句

return语句终止当前正在执行的函数并将控制权返回到调用该函数的地方。

return语句返回一个值的方式和初始化一个变量或者形参的方式完全一致，返回的值用于拷贝初始化调用点的一个临时量，这个临时量就是函数调用的结果。

6.4.1 返回引用类型的函数

与一般的引用类型一样，返回引用类型的函数，返回值是被引对象的一个别名。指的是该函数的返回值是被引对象的一个引用，直接绑定与该被引对象。

该函数完成的功能是比较两个string类型的形参的长短，挑出较短的那个并返回其引用。 函数的返回值是绑定在s1和s2较短那个的引用。

注意：返回引用的函数，返回值必须是绑定在函数外部定义的已经存在的对象。不能尝试返回局部变量的引用，因为函数结束时局部变量占用的空间会随之释放。

6.4.2 列表初始化返回值

函数可以返回花括号包围的值的列表，此处的列表用于对函数返回的临时变量进行初始化，如果列表为空，返回的值由函数的返回类型决定。

如果函数返回的是内置类型，用于列表初始化的列表最多包含一个值，如果返回的是类类型，由类本身定义初始值应该如何使用。

6.4.3 返回数组指针

因为数组不能被拷贝，所以函数不能返回数组。

但是函数可以返回数组的指针或者引用来间接返回数组类型。

6.5 函数重载

函数重载指的是在同一作用域中，几个函数名字相同但形参列表不同。这些函数接受的形参列表不一样，但是执行的操作非常类似。调用这些函数时，编译器会根据传递的实参类型推断想要的是哪个函数。

注意：形参的名字只起到记忆作用，实际上判断两个重载函数的形参列表是否一致，只判断对应的形参的类型。

6.6 默认实参

在函数定义的时候，可以为形参制定一些默认值作为默认实参。和Python相似，使用默认实参的时候只需要在调用函数的时候忽略该实参就可以了。因此设计含有默认实参的函数，需要合理调整参数顺序，将经常使用默认值的形参出现在后面，方便直接忽略。

通常应当在函数声明中指定默认实参，并将该声明放在合适的头文件中。

局部变量不能作为默认实参，除此之外，只要表达式的类型能够转换成形参所需的类型，该表达式就可以作为默认实参。

6.7 调试帮助

6.7.1 assert预处理宏

assert宏使用一个表达式作为它的条件，如果表达式为假，assert输出信息并终止程序运行，如果为真，则assert什么也不做。

assert常用于检查不能发生的条件。

Intra-Inter Camera Similarity for Unsupervised Person Re-Identification

发表于2022-03-15更新于2022-03-18

CVPR2021 Intra-Inter Camera Similarity for Unsupervised Person Re-Identification 学习笔记

研究问题的提出

作者提出，现有的无监督行人再识别工作主要可以分为3类：a)使用领域自适应的方法来对齐源域和目标域之间的特征分布。b)应用GAN网络执行图像风格迁移并维护源域上的身份标注。3）通过聚类算法、KNN等算法对图像进行聚类，在目标域上生成伪标签进行训练。前两者将Re-ID任务看作是一个迁移学习的任务，需要利用源域上的标注。而基于伪标签生成的ReID方法可以在完全无监督的情况下训练ReID模型，从而具有更好的灵活性。

目前大多数基于伪标签生成的ReID方法都是先计算样本相似度，然后通过聚类等算法把相似的样本分配给相同的伪标签。在此过程中计算的样本相似度在很大程度上决定了ReID的准确性。但是现有的无监督ReID方法很难学习到可靠的样本相似度，特别是来自于不同相机的样本，由于相机之间的参数设置不同，这些因素可能会显著影响图片中人的外观，从而直接对全部的图片进行聚类，不考虑相机之间的领域差距，无法生成高质量的伪标签，从而影响ReID的性能和精度。

作者提出的方法

行人再识别的任务是在未标记的任务图像数据集上，选取一张人物图像q，输入到ReID模型中产生一个特征向量，以从图库集中检索包含同一个人的图像Ig。模型应保证q和Ig比q和其他图片共享更多相似特征。

3 AIBN

首先为什么需要归一化（规范化层）？

数据预处理的方式通常会对最终结果产生巨大影响。在训练刚开始的时候我们一般需要标准化输入特征，使其平均值为0，方差为1。即尽量使数据的分布在正态分布上，这样的标准化可以很好的与我们的优化器配合使用，它可以将参数的量级进行统一。

第二，对于典型的多层感知机或卷积神经网络。当我们训练时，中间层中的变量（例如，多层感知机中的仿射变换输出）可能具有更广的变化范围：不论是沿着从输入到输出的层，跨同一层中的单元，或是随着时间的推移，模型参数的随着训练更新变幻莫测。 批量规范化的发明者非正式地假设，这些变量分布中的这种偏移可能会阻碍网络的收敛。因此我们需要在每次训练迭代中，对数据输入进行规范化处理，以加速网络的收敛。

更深层的网络很复杂，容易过拟合。 这意味着正则化变得更加重要。

Normalization 归一化

3.1 Instance Normalization 实例归一化

IN常常用在图像风格化中，图像风格化生成的结果依赖于某个图像实例，因此对输入的规范化处理应该基于每个样本自身的分布。因此是在单一样本中对HW维度做归一化。

3.2 Batch Normalization 批量归一化

直观上理解就是BN和IN都是做Normalization，BN是对一个batch上的全部特征图在通道上计算均值方差做归一化操作。

3.3 AIBN作者提出的Adaptive Instance and Batch Normalization

对样本特征的归一化处理，结合实体和batch两个层面的考虑。实验结果表明其有效性

实现细节

使用ResNet-50作为主干网络来提取特征， 在训练过程中，输入图像的大小被调整为 256×128。执行随机翻转和随机擦除等图像增强策略。在每一轮中，我们依次执行相机内阶段和相机间阶段。训练轮数设置为 40。

在相机内训练阶段，每个相机的batch大小为 8。 使用SGD来优化网络模型。ResNet-50 基础层的学习率为 0.0005，其他层的学习率为 0.005。

在相机间训练阶段，使用 P = 16 个随机选择的簇和 每个簇K =4 个随机采样的图像。mini-batch设置为64。也使用 SGD 优化模型。 ResNet-50 基础层的学习率为 0.001，其他层的学习率为 0.01。相机间损失函数方程中的损失权重λ固定为 1。triplet-loss 的Margin固定为 0.3。训练逐渐统一来自不同相机的特征分布。

对于 Market1501 和 DukeMTMC-ReID，我们在每个阶段都训练模型 2 个 epoch。对于 MSMT17，我们在相机内阶段训练模型 12 个 epoch，在相机间阶段训练 2 个 epoch。我们使用standard Agglomerative
Hierarchical method聚类方法进行聚类。对于 Market1501 和 DukeMTMC-ReID，相机内阶段每个相机的cluster数量设置为 600，相机间阶段为 800。对于 MSMT17，相机内阶段每个相机的cluster数量为 600，相机间阶段为 1200。尽管在每个相机内执行了额外的聚类，但这比在整个集合上聚类更有效。因此，我们方法的计算复杂度是可以接受的。在 Market1501 上使用 GPU 完成训练大约需要 4-5 小时。对于 AIBN，混合权重 α 初始化为 0.5。我们将 ResNet-50 的 layer3 和 layer4 中的所有 BN 替换为 AIBN。每个 BottleNeck 模块共享混合权重。

DLA网络模型：一种深度网络特征融合方法

DLA简介

DLA网络模型出自CVPR 2018 的论文 Deep Layer Aggregation，中文翻译为深层聚合。CNN为多种计算机视觉任务提供了很好的解决方案。随着视觉任务对高性能算法更严格的追求，Backbone的设计成为了一个很重要的主题。作者在摘要中指出计算机视觉识别任务需要从低到高的层次提取不同层次的信息。因此当前的网络设计方向是如何加宽加深网络层次以提取更多的特征。如ResNet，通过引入跳跃连接使得网络有能力到达很深的层次。设计更深或更宽的网络架构虽然确实可以更好的提取特征，但也带来了更多的参数数量和难优化问题。更深的层提取更多语义和更全局的特征，但这些并不能证明最后一层是任何任务的最终表示。事实上，skip connections已经被证明对于分类和回归以及更结构化的任务是有效的。如何更好的聚合不同层次提取的特征，就像深度和宽度一样，是网络架构的一个关键维度。因此作者认为：设计更深更宽的网络结构固然重要，但如何更好的聚合层和块更值得研究。

经过复杂的分析，更深的网络层能提取到更多语义和全局的特征，但是这并不能表明最后一层就是任务需要的表示。实际上“跳跃连接”已经证明了对于分类、回归以及其他结构化问题的有效性。因此，如何聚合，尤其是深度与宽度上的聚合，对于网络结构的优化是一个非常重要的技术。

作者认为，在计算机视觉任务中，仅是加深网络层次，但层和层之间相互孤立是不行的。混合和聚合不同层提取的特征表示，可以改进对图像是什么和在哪里的推断。尽管ResNet加入了skip connections的设计，已经组合了不同层次的特征表示，但这些连接本质上是浅层的连接，仅仅通过相加操作来融合不同层次的特征。

作者通过更深层次的聚合来增强标准体系结构，以便更好地跨层融合信息。我们的深层聚合结构迭代和分层地合并特征层次，使网络具有更高的精度和更少的参数。跨架构和任务的实验表明，与现有的分支和合并方案相比，深层聚合方案提高了识别和分辨率。

第一章 DLA的基本组成模块–IDA和HDA

DLA，即深度聚合。在论文中，作者将聚合Aggregation定义为网络架构中不同层次特征表示的组合。多个layer组合为一个block，多个block再根据分辨率组合为一个stage，DLA则主要探讨block和stage的组合（stage间网络保持一致分辨率，那么空间融合发生在stage间，语义融合发生在stage内）。如果一组Aggregation是复合的、非线性的，并且较早的聚合层的输出输入到后面的多个聚合层。作者介绍了两种深层聚合结构：IDA(iterative deep aggregation迭代式深度聚合)和HDA(hierarchical deep aggregation层级深度聚合)。IDA专注于跨分辨率和尺度的融合，而HDA专注于融合所有提取自不同模块和通道的特征。下图1的c和f分别展示了IDA和HDA的结构。

由于一两个月没有搞代码了，刚回家就碰见了个非常脑残小儿科的问题却一直没有解决。

问题描述

回家做一个ResNet50的CIFAR10分类找找之前遗忘的知识，出现了训练集的loss正常下降、准确率正常上升。而测试集的loss却几乎不下降甚至还有反增，准确率保持非常低的水平不变。如下图所示：

可见训练集准确度已经到了很高的程度，但是测试集的loss和准确率几乎没有什么变化。仔细检查代码（指查了三天）发现在数据的预处理部分，训练集做了Resize、随机翻转和均值化操作，但是测试集没有作任何处理。

即问题出在训练集和测试集没有作相同的预处理。引用知乎一个很恰当的比喻是：你复习了一整晚的高数课本，可是第二天考线性代数，虽然二者都属于数学范畴，但是你拿高数知识解线代怎么能做的好呢？对数据集作预处理，如均值化操作，实际上对数据的分布作了改变。训练集训练的是一个数据分布，直接拿训练的网络对另外一个数据分布的测试集作测试，效果肯定不好。

问题解决方法

将数据加载的预处理统一，训练集和测试集使用相同的预处理方式。

测试集准确率就正常上升了。

学习与反思

借用知乎回答“深度学习中为什么要对测试集进行与训练集一样的数据预处理？”

作者：深海
链接：https://www.zhihu.com/question/479694291/answer/2061254022

我们拿标准化举例，标准化也是数据预处理的过程，我们假设将要标准化的某个特征服从高斯分布。

在神经网络中也可以看作是使网络收敛更加稳定的trick（因为标准化后大部分数值落在一定区间内）。

我们只能通过训练集获得到数据中该特征的所有可能的情况，我们假设我们拿到的训练集是足够有代表性的，能够从训练集中尽可能逼近数据本身服从的概率分布（这也就是为什么扩充数据能够提升模型泛化能力，我们拿到的数据越多，越能接近数据本身的分布，如果我们拿到全量数据，那么本身的分布就确定下来了，假设数据的某个特征服从高斯分布，那么它的均值和方差也确定了，如果我们能拿到全量数据，那也就没必要做机器学习了）。

所以我们是拿训练集中数据的某个特征代表了全部数据的这个特征，用它的均值和方差代表了全部数据的均值和方差。这就很容易理解了，测试集要使用训练集的均值和方差进行标准化，不就是在用我们假设的数据特征服从的均值和方差进行标准化吗？

所以不只是深度学习，传统机器学习算法的预处理也需要这么做。深度学习做了标准化后能够对网络的训练有一些益处罢了，这也是为什么做bn，ln的原因，以及网络中用到的各种归一化的trick，对深度学习中网络优化过程中数值的稳定性有一定的帮助。
